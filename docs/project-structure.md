# Структура проекта

Проект организован как инженерный Python-репозиторий для ML-задачи в медицинской предметной области.

## Основные каталоги

- `data/` — рабочие данные проекта. Внутри используются подпапки `raw/`, `interim/` и `processed/`.
- `docs/` — материалы документации `MkDocs`.
- `models/` — локальные артефакты моделей.
- `notebooks/` — исследовательские ноутбуки.
- `src/` — исходный код проекта.
- `tests/` — тесты.

## Основные служебные файлы

- `README.md` — основное описание репозитория и инструкции по запуску.
- `mkdocs.yml` — конфигурация сайта документации.
- `pyproject.toml` — конфигурация Python-проекта и инструментов разработки.
- `requirements.txt` — зависимости проекта.

## Структура каталога src

- `src/data/` — загрузка и подготовка демонстрационного датасета.
- `src/features/` — место для логики генерации признаков.
- `src/models/` — обучение, оценка и управление версиями моделей.
- `src/inference/` — будущие сценарии инференса.
- `src/utils/` — вспомогательные модули, например конфигурация `MLflow`.

Такая структура подходит для учебного ML-проекта и при этом остаётся достаточно близкой к реальной инженерной практике.
